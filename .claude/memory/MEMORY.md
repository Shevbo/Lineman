# Проектная память Lineman — индекс

Долгосрочная память для Claude, работающего в этом workspace. Обязательно к чтению при старте сессии. После любого ключевого изменения — обновлять соответствующий файл (см. триггеры в `CLAUDE.md`).

## Файлы

- [01_architecture.md](01_architecture.md) — компоненты, data flow, file layout
- [02_operations.md](02_operations.md) — запуск, рестарт, бэкап, дашборд, мониторинг
- [03_critical_paths.md](03_critical_paths.md) — что нельзя ломать и почему
- [04_incidents.md](04_incidents.md) — журнал инцидентов (YYYY-MM-DD: симптом → root cause → fix)
- [05_config_reference.md](05_config_reference.md) — справка по `config.json`
- [06_testing.md](06_testing.md) — какие тесты, как запускать, что покрыто

## Правила записи

1. Один файл = одна тема. Если тема новая — заводи новый файл, а не пихай в существующий.
2. Дата в формате `YYYY-MM-DD` всегда.
3. Никаких значений секретов — только имена env-переменных и пути.
4. Если факт устарел — правь/удаляй, не плоди дубли.
5. Связи между файлами — markdown-ссылками `[имя](файл.md)`.
